# Development Process

## Agile Hackathon Workflow

- Work is organized into five phases: architecture, core implementation, integration, verification, demo preparation.
- Each phase has a single owner and shared review points.
- Deliverables are documentation and module artifacts rather than full production code.

## Collaboration Practices
- Use `docs/` and `specs/` to capture decisions before implementation.
- Keep module interfaces stable with contract-first definitions.
- Perform daily syncs on blocker status and integration dependencies.

## Review and Handoff
- Architecture review after Phase 1.
- Module review after Phase 2.
- Integration review after Phase 3.
- Demo readiness review after Phase 5.
