# SPEC_FRONTEND_DASHBOARD

## 1. Purpose
Define the frontend dashboard subsystem for Sentinel AI, including user workflows, UI contracts, and data integration with the backend.

## 2. Scope
- Provide a user interface for domain submission, scan status, findings review, remediation actions, and report access.
- Consume backend REST API endpoints and present structured results.
- Exclude backend orchestration, scanner operations, and report generation logic.

## 3. Owner
PERSON C

## 4. Technologies Used
- React
- JavaScript / TypeScript
- Fetch / Axios (HTTP client)

## 5. Inputs
- User domain input
- Scan profile selection
- Task status payloads from backend
- Findings and AI analysis payloads from backend
- Remediation action responses
- Report URL metadata

## 6. Outputs
- Scan initiation requests
- Fix and Fix All requests
- Report generation requests
- UI state updates and user notifications

## 7. JSON Contracts
### Scan initiation request
{
  "domain": "example.team-owned-site.com",
  "scanProfile": "full-scan"
}

### Status response
{
  "taskId": "task-1234",
  "status": "analysis",
  "currentStep": "ai_analysis",
  "progress": 60
}

### Results response
{
  "taskId": "task-1234",
  "domain": "example.team-owned-site.com",
  "reconData": {...},
  "findings": [
    {"findingId": "f-001", "title": "Missing security headers", "severity": "medium", "description": "...", "recommendationId": "r-001"}
  ],
  "aiAnalysis": {...},
  "reportReady": false
}

### Fix request
{
  "action": "fix",
  "recommendationId": "r-001"
}

### Report response
{
  "taskId": "task-1234",
  "reportUrl": "http://localhost:8000/reports/report-1234.pdf",
  "artifactPath": "/app/reports/report-1234.pdf"
}

## 8. Interfaces
- Component: `ScanForm`
- Component: `TaskStatusPanel`
- Component: `FindingsList`
- Component: `AiAnalysisCard`
- Component: `RemediationActionBar`
- Service: `apiClient.fetchScanStatus(taskId)`
- Service: `apiClient.fetchScanResults(taskId)`
- Service: `apiClient.submitFixAction(taskId, action, recommendationId)`
- Error boundary: `ApiErrorBoundary`

## 9. Error Handling
- Display validation errors from backend when domain input is invalid.
- Show appropriate messages for target not allowed or forbidden responses.
- Provide retry options for transient backend failures.
- Show a user-facing fallback if AI analysis is unavailable.
- Keep UI responsive during long-running scan operations.

## 10. Testing Strategy
- Unit tests for individual React components and state logic.
- API contract tests using mocked backend responses.
- UI workflow tests for scan submission, status polling, and remediation actions.
- Accessibility checks for critical dashboard screens.
- Regression tests for report link display and error states.

## 11. Acceptance Criteria
- User can submit a controlled target domain and select scan profile.
- Dashboard displays scan progress and current step accurately.
- Findings and AI recommendations are clearly presented.
- Fix and Fix All interactions call the correct backend endpoints.
- Final report link appears when report generation succeeds.

## 12. Integration Requirements
- Consume backend contracts from `SPEC_FASTAPI_ORCHESTRATION.md`.
- Display findings and AI output shaped by `SPEC_AI_ANALYSIS.md`.
- Show remediation statuses produced by `SPEC_REMEDIATION_AGENT.md`.
- Provide report URL access generated by `SPEC_FORENSIC_REPORT.md`.
- Maintain stable UI contract boundaries so backend changes do not break the dashboard.
